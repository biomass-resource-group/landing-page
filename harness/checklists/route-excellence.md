# Route excellence criteria

What "exceptional" (≥4.5) looks like for each route. Use this as a
reference during `/optimize`, `/polish`, and convergence loops.

## `/` (Homepage)

- **Hero**: Single H1 in Cormorant Garamond. Eye reaches headline
  before any other element. ≤2 CTAs. Metric strip below.
- **Sections**: 3–5 sections with clear eyebrow → heading → body flow.
  Alternating plain/tinted backgrounds. One dark section max.
- **Visual**: Generous whitespace. No visual clutter. Gold used
  sparingly as accent. Forest for primary CTAs.
- **Copy**: Every sentence quantitative or specific. No superlatives.
  Hero headline ≤16 words.
- **Performance**: LCP < 1.5s. No layout shift. Fonts loaded with
  swap.

## `/platform/`

- **Hero**: Clear value proposition. "What the platform does" in one
  sentence.
- **Sections**: Technology → process → output flow. Each section
  self-contained with one key concept.
- **Visual**: Diagrams or structured data preferred over prose walls.
  Cards for feature comparison.
- **Copy**: Technical but accessible. Avoid jargon without context.

## `/markets/`

- **Hero**: Geographic scope statement.
- **Sections**: Market cards with consistent structure (region, status,
  key metric). Grid layout.
- **Visual**: Cards uniform in size and information density.
- **Copy**: Specific country/region names. "Operating" vs "pipeline"
  clearly distinguished.

## `/about/`

- **Hero**: Mission/identity statement.
- **Leadership**: Bio cards with photo, name, role, summary.
  Consistent card sizes. Julie Brown has LinkedIn link. Cody Danet
  does NOT have LinkedIn link.
- **Visual**: Professional, warm. Photos (when present) same aspect
  ratio and treatment.
- **Copy**: Operator-grounded identity. Track record over aspiration.

## `/contact/`

- **Hero**: Clear "get in touch" message.
- **Form**: Clean, accessible form with proper labels, focus states,
  error handling.
- **Visual**: Simple. No visual noise competing with the form.
- **Copy**: Brief. What happens after they submit.

## `/company/`

- **Redirect**: Should redirect to `/about/`. If it renders, ensure
  it matches `/about/` quality.

## Cross-route excellence

- Same spacing rhythm everywhere.
- Same typography scale everywhere.
- Same CTA style everywhere.
- Same eyebrow treatment everywhere.
- Footer identical on all routes.
- Header + nav identical on all routes.
- Mobile experience matches desktop quality.
